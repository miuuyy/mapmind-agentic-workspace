# Community

Clew is still a small personal product. If the workspace feels useful, unusual, or worth pushing further, the best thing you can do is stay close to it.

## Follow the project

- Star the repository: [github.com/miuuyy/mapmind-agentic-workspace](https://github.com/miuuyy/mapmind-agentic-workspace)
- Open an issue if something feels broken, unclear, or surprisingly good
- Send a pull request if you want to improve the workspace directly

## What kind of contributions fit

Useful contributions are usually concrete and product-facing:

- graph UX improvements
- documentation improvements
- provider integrations
- import and export improvements
- topic, quiz, or study-flow quality fixes
- architecture cleanups that preserve the product stance

The repository already includes project-specific skills, a partially prepared CI-agent workflow, and readable internal docs, so contributing does not start from chaos.

If you want to improve the project with CI-agent help instead of manual repo archaeology, the repo is already shaped to make that practical.

## Feedback is valuable even without code

If you do not want to contribute code, feedback still matters:

- tell me what confused you
- tell me what felt powerful
- tell me what looked promising but did not quite land
- tell me what subjects or workflows you would want to use this for

That feedback is useful because the product is still being shaped in public.
