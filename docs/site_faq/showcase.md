# Showcase

A few concrete cases that show what Clew is useful for.

## 1. A messy goal becomes a path

You start with something vague:

> I want to learn enough math for machine learning.

Normally that becomes tabs, videos, saved posts, and a guilt pile.

In Clew, AI can draft the structure: foundations, dependencies, middle layers, advanced branches, and frontier topics. You inspect the proposal before it becomes part of the graph.

The point is not that the graph is pretty. The point is that the subject becomes legible.

## 2. A topic shows its neighborhood

Click a topic and you see:

- what leads into it
- what it unlocks
- what is blocked
- which region it belongs to
- where your current frontier sits

A topic list turns into a usable route.

## 3. Obsidian notes become structure

If you already have an Obsidian vault, Clew can import it.

Markdown notes become topics. Links become graph relationships. Folders can become zones. Missing links can become placeholders when you want them.

Existing notes turn into a map you can study from.

## 4. The graph goes back to Obsidian

Clew does not lock your data in.

You can export a graph as an Obsidian-ready folder with topic pages, resources, artifacts, and metadata. Obsidian stays the long-term vault; Clew is where the learning path is shaped.

## 5. External assistants can see the workspace

With Clew Study Assist MCP, Claude Desktop, Claude Code, Cursor, or another MCP client can inspect your local learning graph.

You can ask:

- what am I blocked on?
- what should I study next?
- what topic connects these two ideas?
- what notes do I already have about this?

The MCP server is read-only. External assistants can see context but cannot change the graph.

## 6. Progress stays attached to the graph

Clew supports closure quizzes so completion can mean more than clicking "done".

For lighter workflows, strict closure can be disabled and topics can be marked finished manually. The point is not to force school behavior — progress lives on the path, not on a separate checklist.
